# Bambu AI Print Monitor

Home Assistant 自定义集成，通过通义千问（Qwen-VL）视觉模型实时监测拓竹（Bambu Lab）3D 打印状态。

## 功能

- **AI 打印监测**：通过拓竹打印机摄像头截取快照，发送给通义千问 AI 分析打印质量
- **异常检测**：自动识别以下打印异常：
  - 炒面/面条状挤出（Spaghetti）
  - 翘边（Warping）
  - 层偏移（Layer Shift）
  - 欠挤出（Under Extrusion）
  - 过挤出（Over Extrusion）
  - 脱落（Detachment）
- **自动暂停**：检测到异常时自动暂停打印（可配置连续确认次数，防止误报）
- **实时状态**：打印进度、温度、剩余时间、层进度等
- **手动控制**：暂停/恢复/停止打印按钮，立即触发分析
- **多打印机支持**：可同时添加多台打印机

## 安装

### 通过 HACS 安装

1. 将本仓库添加到 HACS 自定义仓库
2. 在 HACS 中搜索 "Bambu AI Print Monitor" 并安装
3. 重启 Home Assistant

### 手动安装

1. 将 `custom_components/bambu_ai_monitor` 文件夹复制到 Home Assistant 的 `custom_components` 目录
2. 重启 Home Assistant

## 配置

### 前提条件

1. **拓竹打印机**：X1C、X1E、P1P、P1S、A1、A1 Mini 系列，启用 LAN 模式
2. **通义千问 API Key**：在 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/) 注册并获取
3. **打印机 LAN Access Code**：在打印机屏幕上获取

### 添加集成

1. 进入 Home Assistant → 设置 → 设备与服务 → 添加集成
2. 搜索 "Bambu AI Print Monitor"
3. 填写打印机连接信息：
   - 打印机 IP 地址
   - LAN Access Code
   - 打印机型号
   - 摄像头端口（默认 6000）
4. 填写 API Key 和选择 AI 模型
5. 配置分析选项：
   - 分析间隔（30秒 ~ 30分钟，推荐 5 分钟）
   - 置信度阈值（0.1 ~ 1.0，推荐 0.7）
   - 是否启用异常自动暂停
   - 连续检测次数（推荐 2 次，防止误报）

## Debug 模式（模拟打印机）

在没有真实打印机的情况下，集成为开发者提供了内置的模拟打印机功能：

1. **IP 地址**：填写 `mock` 或 `127.0.0.1`
2. **Access Code**：填写 `DEBUG_MODE`（必须完全一致）
3. 其他字段按正常填写即可

模拟打印机会：
- 模拟打印进度从 0% 到 100% 的过程（约 1 小时）
- 在 25%-35% 和 65%-75% 进度时模拟"炒面"和"翘边"异常
- 生成模拟的摄像头快照图片（正常时绿色方块，异常时散乱像素）
- 支持暂停/恢复/停止命令
- 所有日志会带有 `[DEBUG MODE]` 前缀

> **注意**：Debug 模式仍会调用通义千问 API 分析模拟图片，会产生少量 API 费用。

## 费用说明

每次 AI 分析会调用通义千问 API，产生少量费用。建议：
- 使用 5 分钟或更长的分析间隔
- 仅在打印中时进行分析（集成会自动判断）
- `qwen-vl-max-latest` 模型性价比高

## 隐私说明

摄像头快照会发送至通义千问云端 API 进行分析。如果您介意打印内容隐私，请谨慎使用。

## 故障排查

### 打印机连接失败
- 确认打印机已启用 LAN 模式
- 确认 Access Code 正确
- 确认防火墙未阻止 8883 端口

### AI 分析失败
- 确认通义千问 API Key 有效且有余额
- 检查 Home Assistant 日志
- 确认摄像头端口可达（默认 6000）

### 频繁误报
- 提高置信度阈值（建议 0.7-0.85）
- 增加连续检测次数（建议 2-3 次）
- 检查摄像头画面是否清晰
